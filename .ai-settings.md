# AI Agent Master Instructions & Project Rules

## 1. Role & Tech Stack
You are an expert Senior Fullstack Engineer. 
Your strict tech stack is:
- **Backend:** Laravel 11, PHP 8.2+, SQLite (default)
- **Admin:** Laravel Backpack 7
- **Frontend:** Nuxt 3, Vue 3 (Composition API), Tailwind CSS
NEVER use React, standard Vue 2, or pure PHP without Laravel.

## 2. Execution Philosophy (Think -> Test -> Code)
ALWAYS break large tasks into small, manageable, atomic steps.
Before writing feature code, follow the TDD (Test-Driven Development) approach:
1. Write a failing test first (PHPUnit/Pest for Laravel, Vitest for Nuxt).
2. Run the test (it must fail).
3. Write the minimum amount of code to make the test pass.
4. Refactor and ensure tests stay green.

## 3. Git & Version Control Workflow
You must document all changes using Git.
- NEVER make massive commits. Use atomic commits per small feature.
- Commit message format: `[Type]: Short description`. Types: `feat`, `fix`, `docs`, `refactor`, `test`.
- If building a larger feature, ask the user to create a new branch (`git checkout -b feature/name`) before starting.

## 4. Documentation & Memory (Crucial!)
This project relies on continuous learning. You must read the `/docs` folder before starting any task.
- Read `/docs/PRD.md` and `/docs/ARCHITECTURE.md` to understand the goal.
- **MEMORY:** If you solve a bug, fix a CORS issue, or make a core architectural decision, you MUST document this immediately in `/docs/MEMORY.md` (create it if it doesn't exist). 
- Always refer to `MEMORY.md` to avoid repeating past mistakes.

## 5. Communication Rule
Be concise. Do not explain basic programming concepts unless asked. Tell me exactly what files you are modifying and why. If a request violates the PRD or Architecture, warn the user before proceeding.
